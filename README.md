# Welcome to Rishabh Rohira's Digital Web Portfolio!

## What is this project?

- This project serves as a digital collection of my most impressive web development work.
- It debriefs recruiters and clients on what I can do, who I am, and the mindset I employ when making choices.

## Why a web portfolio?

- I learned web development in class, so it only makes practical sense to showcase impressive work in an impressive way on a website.
- If web development work can be viewed globally, it doesn't make sense to have a local PDF or a physical portfolio of digital work.

## What's your tech stack?

- React and Tailwind CSS for frontend logic
- There's no backend
- Prettier for tooling and formatting
- Github deployment temporarily, likely to move elsewhere

## What features did you add?

- Users can directly access all the projects I've worked on, as they're all open source
- The website is largely accessible for screen readers and colorblind users
- The website runs fast and is largely optimized for performance
- There are cursor animations, heading animations, and scroll animations
- The website has a contact section to directly reach me via email
- The newspaper styling of the website makes it incredibly unique and eye catching to users who have seen numerous bland site styles

## What does the website's structure look like?

- It runs on the original React folder structure, with "src" files controlling the frontend
- 3 .jsx files: Hero.jsx, main.jsx, and TargetCursor.jsx
- The web app is statically generated
- This format is the most simple, since I want to focus on the content more than the structure

## Why React?

- I chose React because of the large online community available to help me out with bugs and learning
- React allows for highly reusable code, and it performs really well with single page applications like this portfolio
- React has a lower barrier to entry and is a high performance framework

## Why this styling approach?

- I prefer calmer, more muted tones over sharp, bright colors
- I didn't want to do dull gray tones, and nor did I want to have a flashy portfolio that has more cool animations than concrete proof of work.
- I planned on incorporating dark, amber, and bright colors together, however the variety inevitably lacked depth. I value depth.
- Moving on, I definitely plan on having hands-free access to scrolling on the website and anchored scroll effects.

## What lessons did you learn?

- I couldn't decide on the best way to showcase projects. Some were clunky, and others ugly. The tradeoff was to be clear with my work before decorating it too much.
- Moving forward with web development, I plan to always have foundations for aesthetically pleasing work, but the first focus should always be either accessibility or clarity from now on.
- This portfolio project strengthened my understanding of what users are looking for. Once I can understand my audience, I can tailor to their needs and expectations of the website.
